3 C$$$ SUBPROGRAM DOCUMENTATION BLOCK
4 C . . . .
5 C SUBPROGRAM: W3TRNARG TRANSLATES ARG LINE FROM STANDARD INPUT
6 C PRGMMR: KEYSER ORG: NP22 DATE: 2002-02-11
7 C
8 C ABSTRACT: READS ARGUMENT LINES FROM STANDARD INPUT AND OBTAINS ,
9 C SUBDIRECTORY, BUFR TANKNAME, CHARACTERS TO APPEND FOR ADDING
10 C AN ORBIT, AND OPTIONS FOR LIMITING THE TIME WINDOW.
11 C
12 C PROGRAM HISTORY LOG:
13 C 1996-09-03 B. KATZ -- ORIGINAL AUTHOR
14 C 1998-11-27 B. KATZ -- CHANGES FOR Y2K AND FORTRAN 90 COMPLIANCE
15 C 2002-02-11 D. KEYSER -- IF "TLFLAG" IS NOT SPECIFIED, IT DEFAULTS
16 C TO "NOTIMLIM" RATHER THAN "TIMLIM" AND
17 C GROSS TIME LIMITS WILL NOT BE CALCULATED
18 C AND RETURNED IN "IYMDHB" AND "IYMDHE"
19 C
20 C USAGE: CALL W3TRNARG(SUBDIR,LSUBDR,TANKID,LTNKID,APPCHR,LAPCHR,
21 C TLFLAG,IYMDHB,IYMDHE,IERR)
22 C OUTPUT ARGUMENT LIST:
23 C SUBDIR - NAME OF SUB-DIRECTORY INCLUDING BUFR DATA TYPE WHERE
24 C BUFR DATA TANK IS LOCATED.
25 C LSUBDR - NUMBER OF CHARACTERS IN 'SUBDIR'.
26 C TANKID - NAME OF FILE INCLUDING BUFR DATA SUB-TYPE CONTAINING
27 C BUFR DATA TANK.
28 C LTNKID - NUMBER OF CHARACTERS IN 'TANKID'.
29 C APPCHR - CHARACTERS TO BE APPENDED TO 'TANKID' GIVING A
30 C UNIQUELY NAMED FILE TO CONTAIN THE ORIGINAL TANK
31 C WITH ONE ORBIT APPENDED TO IT.
32 C LAPCHR - NUMBER OF CHARACTERS IN 'APPCHR'.
33 C TLFLAG - 8 CHARACTER FLAG INDICATING WHETHER TIME ACCEPTANCE
34 C CHECKS ATRE TO BE PERFORMED.
35 C = 'TIMLIM ' : PERFORM TIME ACCEPTANCE CHECKS.
36 C = 'NOTIMLIM' : DO NOT PERFORM TIME ACCEPTANCE CHECKS.
37 C JDATE AND KDATE ARE DISREGARDED.
38 C IYMDHB - START OF TIME ACCEPTANCE WINDOW, IN FORM YYYYMMDDHH.
39 C IYMDHE - END OF TIME ACCEPTANCE WINDOW, IN FORM YYYYMMDDHH.
40 C
41 C INPUT FILES :
42 C UNIT 05 - STANDARD INPUT FOR PASSING IN ARGUMENTS. ARGUMENTS
43 C (FOR LIST-DIRECTED I/O) ARE AS FOLLOWS :
44 C RECORD 1 - (1) SUBDIRECTORY. CONTAINS BUFR DATA TYPE
45 C (2) TANKFILE. CONTAINS BUFR DATA SUB-TYPE
46 C (3) APPEND CHARACTERS. APPENDED TO TANKFILE
47 C TO GIVE UNIQUE OUTPUT FILE NAME.
48 C (4) DATE IN YYYYMMDDHH FORMAT.
49 C NEXT THREE RECORDS ARE OPTIONAL :
50 C RECORD 2 - (1) TIME LIMIT FLAG. MAY BE EITHER
51 C 'TIMLIM ' OR 'NOTIMLIM'. SEE
52 C DESCRIPTION OF 'TLFLAG' ABOVE.
53 C (DEFAULT IS 'NOTIMLIM')
54 C RECORD 3 - (1) HOURS BEFORE CURRENT TIME.
55 C RECORD 4 - (1) HOURS AFTER CURRENT TIME.
56 C IF 'TIMLIM ' IS SPECIFIED IN RECORD 2, THE
57 C QUANTITIES IN RECORDS 3 AND 4 ARE USED TO
58 C COMPUTE THE LIMITS OF THE TIME ACCEPTANCE WINDOW.
59 C IF RECORDS 3 AND 4 ARE OMITTED, THE VALUES
60 C DEFAULT TO -48 (48 HOURS BEFORE CURRENT TIME)
61 C AND +12 (12 HOURS AFTER CURRENT TIME).
62 C IF 'NOTIMLIM ' IS SPECIFIED IN RECORD 2, THEN
63 C THESE QUANTITIES ARE NOT USED REGARDLESS OF WHETHER
64 C OR NOT THEY WERE SPECIFIED.
65 C
66 C SUBPROGRAMS CALLED :

68 C
