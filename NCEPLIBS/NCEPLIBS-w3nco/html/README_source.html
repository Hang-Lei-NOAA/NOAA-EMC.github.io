8  README.w3
9 
10  This directory, /nwprod/lib/sorc/w3 on IBM RS/6000 SP, contains various
11  relocatable libraries. Several versions of each library may exist
12  and they follow the naming convention libname_n_xxx.
13 
14  The first qualifier in the library name, _n, is set to _4, _d or
15  _8 to indicate the default size of the integer and real variables used
16  in the routines. Libraries with the _4 qualifier were compiled
17  specifying that integer and real variables have a default size
18  of 4 bytes, whereas the _8 libraries compiled the source specifying
19  8 byte integer and real variables (i.e. compiler options "-qintsize=8"
20  and "-qrealsize=8" were used). The third type, _d, indicates that
21  real variables are 8 bytes, but integer variables are 4 bytes
22  (i.e. compiler option "-qrealsize=8" is used). Note that
23  "-qintsize=4" and "-qrealsize=4" are the default compiler options.
24 
25  The second qualifier indicates the RISC processor type on which the
26  library was compiled to run. The _604 qualifier indicates that the
27  library was compiled to run on the 604 processor used in the Silver
28  Nodes ( compiler option "-qarch=604" ). The _pwr3 libraries were
29  compiled using option "-qarch=pwr3" to run on the Winterhawk Nodes.
30  Note that codes compiled with "-qarch=604" will run on the PWR3
31  processors, but programs specifying "-qarch=pwr3" may not run
32  properly on the 604 processors.
33

113 
114  Library Maintenance
115 
116 
117  * "makew3lib.sh", "makesplib.sh", "makeiplib.sh", "makegemlib.sh"
118  and "makebacio.sh" are used primarily to compile source code
119  files and to place their object files into libraries "w3lib",
120  "splib", "iplib", "gemlib", and "bacio", respectively.
121 
122  * The compile options used in the scripts mentioned above are "-O3"
123  "-qnosave", "-qarch=604 or pwr3". Options "-qintsize=8" and/or
124  "-qrealsize=8" are used to create the _d or _8 libraries.
125 
126  * The "archive ar" command is also used in the compile scripts. It
127  enables the user to manipulate the relocatable libraries
128  in a number of ways.
129 
130  * "ar -r" adds a module to the relocatable library or replaces
131  the module in the library if it already exists.
132 
133  ex. ar -r w3lib_8_604 w3fi88.o
134 
135  * "ar -d" deletes the named module from a relocatable library.
136 
137  ex. ar -d w3lib_8_604 w3fi88.o
138 
139  * "ar -tv" prints a table of contents and gives a verbose module by
140  module description of the making of a new library file. This is
141  useful to do after running the compile script to verify how
142  the relocatable library has been altered.
143 
144  ex. ar -tv w3lib_8_604
145 
146 
147  NOTE: 1) It is a good idea to make a backup copy of the relocatable
148  library before manipulating it in any way.
149 
150  2) The script uses the make utility to add or to update all w3lib
151  archive libraries: w3lib_4_604, w3lib_4_pwr3, w3lib_8_604,
152  w3lib_8_pwr3, w3lib_d_604, w3lib_d_pwr3, w3mod_4, w3mod_8,
153  and w3mod_d.
154 
155  For example, you would enter the following:
156 
157  /nwprod/lib/sorc/w3/makew3lib.sh
158 
159 
160  REMARKS: Only source files that have been modified since
161  the last library update are recompiled and replaced
162  in the object archive libraries. The make utility
163  determines this from the file modification times.
164 
165  New source files are also compiled and added to the
166  object archive libraries.
167 
168  3) If you need to re-compile all object files, you have to change
169  the file date by using the command "touch *.f " or rename the
170  existing libraries "w3lib". Then, you run the script
171  "makew3lib.sh" to generate all w3lib archive libraries.
