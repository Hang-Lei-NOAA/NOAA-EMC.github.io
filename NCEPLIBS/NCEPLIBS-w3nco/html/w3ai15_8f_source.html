7 C ABSTRACT: CONVERTS A SET OF BINARY NUMBERS TO AN EQUIVALENT SET

18 C
19 C USAGE: CALL W3AI15 (NBUFA,NBUFB,N1,N2,MINUS)
20 C
21 C INPUT:
22 C 'NBUFA' - INPUT ARRAY (INTEGER*4)
23 C ' N1' - NUMBER OF INTEGERS IN NBUFA TO BE CONVERTED
24 C ' N2' - DESIRED CHARACTER WIDTH OF ASCII NUMBER FIELD
25 C 'MINUS' - CHARACTER TO BE INSERTED IN THE HIGH ORDER POSITION
26 C OF A NEGATIVE NUMBER FIELD
27 C OUTPUT:
28 C 'NBUFB' - OUTPUT ARRAY (INTEGER*4)

40 C NOTE 1. - IF N2 IS GREATER THAN 4, ALLOW TWO WORDS (EIGHT CHARACTERS)
41 C IN THE NBUFB ARRAY FOR EACH ASCII NUMBER FIELD. A NUMBER FIELD
42 C IS LEFT ADJUSTED WITH BLANK FILL TO THE RIGHT IF NEEDED.
43 C LIKEWISE, IF N2 IS LESS THAN 4, THE RESULT IS LEFT ADJUSTED
44 C WITH BLANK FILL TO THE RIGHT.
45 C
46 C NOTE 2. - N2 CAN BE SPECIFIED IN THE RANGE 1-8. AN EIGHT DIGIT POSI-
47 C TIVE INTEGER CAN BE CONVERTED OR A SEVEN DIGIT NEGATIVE INTEGER
48 C AND A SIGN. ZERO FILL IS USED FOR HIGH ORDER POSITIONS IN A
49 C NUMBER FIELD. THE USER SHOULD BE AWARE THAT W3AI15 DOES NOT
50 C VERIFY THAT THE VALUE OF N2 IS IN THE CORRECT RANGE.
51 C
52 C NOTE 3. - THE MINUS SIGN CAN BE INSERTED AS A LITERAL IN THE CALL
53 C SEQUENCE OR DEFINED IN A DATA STATEMENT. 1H- AND 1H+ ARE THE
54 C TWO MOST LIKELY NEGATIVE SIGNS. UNFORTUNATELY THE ASCII PLUS
55 C CHARACTER IS THE NEGATIVE SIGN REQUIRED IN MOST TRANSMISSIONS.
56 C THE MINUS SIGN WILL ALWAYS BE IN THE HIGH ORDER POSITION OF A
57 C NEGATIVE NUMBER FIELD.
58 C
59 C NOTE 4. - IF A NUMBER CONTAINS MORE DIGITS THAN THE N2 SPECIFICATION
60 C ALLOWS, THE EXCESS HIGH ORDER DIGITS ARE LOST.
61 C
